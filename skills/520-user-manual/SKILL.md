# 用户使用手册编写

## 描述

编写面向最终用户的使用手册，包括功能说明、操作指南、常见问题和故障排查。

## 使用场景

| 触发条件 | 示例 |
|---------|------|
| 编写用户手册 | "编写用户使用手册" |
| 准备培训材料 | "准备用户培训文档" |
| 整理FAQ | "整理常见问题" |
| 故障排查指南 | "编写故障排查文档" |

## 角色职责

| 角色 | 职责 | 智能体 |
|------|------|--------|
| 主导 | 手册编写 | `test-engineer` |
| 协作 | 功能确认 | `product-manager` |
| 协作 | 界面截图 | `js-developer` |
| 协作 | 完整性确认 | `project-manager` |

## 交付物清单

| 文档 | 格式 | 必需 |
|------|------|------|
| 用户使用手册 | Markdown/PDF | ✅ |
| 快速入门指南 | Markdown/PDF | ✅ |
| FAQ常见问题 | Markdown | ✅ |
| 故障排查指南 | Markdown | ✅ |
| 操作视频 | MP4 | ⚠️ 可选 |

## 文档结构

### 用户使用手册

```
1. 系统简介
   - 系统概述
   - 系统要求
   - 术语说明
2. 快速开始
   - 账号注册
   - 账号登录
   - 界面介绍
3. 功能指南
   - 各模块操作说明
4. 常见问题
5. 故障排查
6. 附录
```

## 编写流程

### 1. 收集资料
- 获取系统功能清单
- 收集界面截图
- 了解目标用户群体

### 2. 编写手册
- 编写系统简介
- 编写快速开始指南
- 编写功能操作指南
- 整理FAQ
- 编写故障排查

### 3. 评审发布
- 团队评审
- 用户试用
- 修改完善
- 归档发布

## 编写原则

| 原则 | 要求 |
|------|------|
| 用户视角 | 避免技术术语，通俗易懂 |
| 步骤清晰 | 操作步骤详细、可执行 |
| 图文并茂 | 配合截图说明 |
| 覆盖全面 | 覆盖所有用户可见功能 |

## PLAN-REVIEW 循环（必须执行）

用户手册编写完成后，必须进入 PLAN-REVIEW 循环，确保文档质量达标。

调用技能 `521-user-manual-review`，评分 ≥ 95 通过，< 95 按 [REVIEW-FIX 循环规范](../0-init/references/review-fix-loop.md) 执行修复循环。

## 输出要求

**存储位置**: `manual/user-manual/`

**目录结构**:
```
manual/user-manual/
├── user-manual.md        # 完整用户手册
├── quick-start.md        # 快速入门指南
├── faq.md                # 常见问题
├── troubleshooting.md    # 故障排查指南
├── videos/               # 操作视频（可选）
└── reviews/              # 评审报告（PLAN-REVIEW循环产出）
    └── REVIEW-USERS-MANUAL-YYMMDDHHMM.md  # 按时间戳命名，24小时制
```
