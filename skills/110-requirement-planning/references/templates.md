# 需求规划模板

## 目录结构

```
requirement/
├── prds/                                  # PRD产品文档
│   ├── README.md                          # 入口文件（项目总览）
│   ├── guest-uniapp/                      # 用户UniApp端
│   │   ├── 0-main.md                      # 项目主文档
│   │   ├── 1-AI对话规划.md                 # 模块文档（数量不限，按需增减）
│   │   └── ...
│   ├── admin-web/                         # 后台管理员Web端
│   │   ├── 0-main.md                      # 项目主文档
│   │   ├── 1-目的地内容管理.md              # 模块文档（数量不限，按需增减）
│   │   └── ...
│   ├── saas-web/                          # 平台管理员Web端
│   │   ├── 0-main.md                      # 项目主文档
│   │   ├── 1-供应商管理.md                 # 模块文档（数量不限，按需增减）
│   │   └── ...
│   ├── mch-web/                           # 商户Web端
│   │   └── ...
│   └── root-web/                          # 系统管理员Web端
│   └── reviews/                               # 评审报告（统一存放）
│       └── REVIEW-PRD-YYMMDDHHMM.md
└── interviews/                            # 访谈记录
    └── INTERVIEW-YYMMDDHHMM-{主题}.md
```

**目录说明**:
| 目录 | 用途 | 命名规则 |
|------|------|---------|
| `prds/` | PRD产品文档，含 README.md 入口文件 | 角色平台子目录：`{角色}-{平台}/` |
| `reviews/` | 评审报告 | `REVIEW-CODE-YYMMDDHHMM.md`，所有项目的评审统一存放 |
| `interviews/` | 访谈记录 | `interview-YYMMDDHHMM.md` |

**角色平台目录命名规则**: `{角色}-{平台}/`，如 `guest-uniapp`、`admin-web`。一个角色对应多个平台时，每个平台独立一个目录（如 `guest-uniapp/` + `guest-web/`）。

**文件命名规则**:
| 文件 | 命名 | 位置 | 说明 |
|------|------|------|------|
| 入口文件 | `README.md` | `prds/` | 项目总览 |
| 项目文档 | `0-main.md` | `prds/{角色}-{平台}/` | 项目级概述 |
| 模块文档 | `{序号}-{中文模块名}.md` | `prds/{角色}-{平台}/` | 序号从1递增，**数量不限** |
| 评审报告 | `REVIEW-CODE-YYMMDDHHMM.md` | `reviews/` | 24小时制时间戳 |
| 访谈记录 | `interview-YYMMDDHHMM.md` | `interviews/` | 24小时制时间戳 |
