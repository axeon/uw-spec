## 4. 实体详细设计

### 4.1 guest_user（用户表）

**实体说明**: C端用户信息

**数据规模**:
| 指标 | 数值 |
|------|------|
| 初始数据量 | 10万 |
| 年增长量 | 50万 |
| 3年后数据量 | 160万 |

**字段定义**:

| 字段名 | 类型 | 长度 | 可空 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | 否 | 自增 | 主键 |
| saas_id | BIGINT | - | 否 | - | 租户ID |
| username | VARCHAR | 100 | 否 | - | 登录名 |
| password | VARCHAR | 100 | 否 | - | 密码(BCrypt加密) |
| mobile | VARCHAR | 20 | 是 | NULL | 手机号(AES加密) |
| email | VARCHAR | 100 | 是 | NULL | 邮箱 |
| create_date | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| modify_date | DATETIME | - | 否 | CURRENT_TIMESTAMP | 修改时间 |
| state | INT | - | 否 | 1 | 状态(0:禁用 1:启用) |

**索引设计**:

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | - |
| uk_gu_mobile | 唯一 | mobile | 手机号唯一 |
| uk_gu_email | 唯一 | email | 邮箱唯一 |
| idx_gu_state | 普通 | state | 状态筛选 |

**非功能设计**:
- 分表策略: 单表，数据量<500万不分表
- 缓存策略: 热点用户数据缓存到Redis，TTL=1小时
- 归档策略: 无

---

### 4.2 order_info（订单表）

**实体说明**: 订单主表

**数据规模**:
| 指标 | 数值 |
|------|------|
| 初始数据量 | 100万 |
| 年增长量 | 1000万 |
| 3年后数据量 | 3100万 |

**字段定义**:

| 字段名 | 类型 | 长度 | 可空 | 默认值 | 说明 |
|--------|------|------|------|--------|------|
| id | BIGINT | - | 否 | 自增 | 主键 |
| saas_id | BIGINT | - | 否 | - | 租户ID |
| guest_id | BIGINT | - | 否 | - | 用户ID(外键) |
| order_no | VARCHAR | 64 | 否 | - | 订单编号 |
| order_state | TINYINT | - | 否 | 0 | 订单状态(0:待支付 1:已支付 2:已取消 3:已完成) |
| order_money | BIGINT | - | 否 | 0 | 订单金额(分) |
| pay_type | TINYINT | - | 是 | NULL | 支付类型(1:微信 2:支付宝) |
| pay_money | BIGINT | - | 是 | NULL | 实付金额(分) |
| pay_date | DATETIME | - | 是 | NULL | 支付时间 |
| create_date | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| modify_date | DATETIME | - | 否 | CURRENT_TIMESTAMP | 修改时间 |
| state | INT | - | 否 | 1 | 状态(0:禁用 1:启用) |

**索引设计**:

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | - |
| uk_oi_order_no | 唯一 | order_no | 订单号唯一 |
| idx_oi_guest_id | 普通 | guest_id | 用户订单查询 |
| idx_oi_state | 普通 | order_state | 状态筛选 |
| idx_oi_create_date | 普通 | create_date | 时间范围查询 |

**非功能设计**:
- 分表策略: 按月份分表，order_info_YYYYMM
- 缓存策略: 最近7天订单缓存到Redis
- 归档策略: 3个月前已完成订单归档到 order_info_archive

| saas_id | BIGINT | - | 否 | - | 租户ID |
