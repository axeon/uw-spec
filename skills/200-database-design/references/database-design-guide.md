# 数据库设计指南

本指南定义数据库设计的规则、规范和最佳实践。
设计使用的数据库类型为 MySQL 8.4+，数据库编码为 UTF8MB4。
---

## 1. 命名规范

### 1.1 核心规则

**表名格式**: `{模块简称}_{实体简称}` — 全小写，两段式

- 模块简称：单个单词，全小写
- 实体简称：驼峰风格转为 snake_case，如 `LoginLog` → `login_log`

| 对象 | 格式 | 示例 |
|------|------|------|
| 表名 | `{module}_{entity_snake_case}` | `guest_user`, `order_info`, `guest_login_log` |
| 主键 | `id` | `id` |
| 字段名 | `snake_case` | `guest_id`, `order_state`, `create_date` |
| 外键字段 | {entity}_id | `guest_id`, `order_id`, `product_id` |
| 普通索引 | `idx_{表名缩写}_{字段名}` | `idx_guest_user_mobile` |
| 唯一索引 | `uk_{表名缩写}_{字段名}` | `uk_guest_user_email` |

| 业务模块 | 简称 | 说明 |
|---------|------|------|
| 用户/访客 | `guest` | C端用户 |
| 管理员 | `admin` | 后台用户 |
| 租户 | `saas` | SaaS租户 |
| 商品 | `product` | 商品管理 |
| 订单 | `order` | 订单交易 |
| 分类 | `cat` | 分类管理 |
| 支付 | `pay` | 支付结算 |
| 消息 | `msg` | 消息通知 |
| 权限 | `auth` | 认证授权 |
| 系统 | `sys` | 系统配置 |
| 内容 | `cms` | 内容管理 |
| 文件 | `file` | 文件存储 |
| 社区 | `post` | 社区分享 |

### 1.3 字段命名规范

#### 通用字段（所有表必须包含）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT UNSIGNED | 主键，自增 |
| `saas_id` | BIGINT | 租户ID（SaaS必加） |
| `create_date` | DATETIME | 创建时间 |
| `modify_date` | DATETIME | 修改时间 |
| `state` | TINYINT | 逻辑状态（0禁用/1启用） |


#### 字段命名规则

| 规则 | 示例 | 说明 |
|------|------|------|
| 布尔字段 `is_` 前缀 | `is_active`, `is_paid` | INT，0=否 1=是 |
| 时间字段 `_date` 后缀 | `create_date`, `pay_date` | DATETIME |
| 价格字段 `_price` 后缀 | `product_price` | BIGINT（分） |
| 金额字段 `_money` 后缀 | `order_money` | BIGINT（分） |
| 数量字段 `_count` 后缀 | `view_count` | INT |
| 外键字段 `_id` 后缀 | `guest_id` | BIGINT |
| 状态字段 `state` | `order_state` | TINYINT，注释注明枚举值 |
| IP 字段 `_ip` | `user_ip` | VARCHAR(50)，IPV4/V6地址 |
| 类型字段 `_type` 后缀 | `pay_type` | TINYINT，注释注明枚举值 |
| 编码字段 `_no` 后缀 | `order_no` | VARCHAR，业务唯一编码 |
| 名称字段 `_name` 后缀 | `user_name` | VARCHAR(100) |
| 标题字段 `_title` 后缀 | `article_title` | VARCHAR(200) |
| 内容字段 `_content` 后缀 | `message_content` | TEXT |
| 图片字段 `_icon` / `_img` / `_url` | `user_icon` | VARCHAR(500) |
| 排序字段 `sort_order` | `sort_order` | INT，从1开始递增 |
| 备注字段 `remark` | `remark` | VARCHAR(500) |
| 扩展字段 `ext_data` | `ext_data` | JSON |
| 多语言字段 `lang` | `lang` | VARCHAR(10)，如 zh-CN, en-US, ja-JP |
| 审核字段 `audit_` | `audit_state`, `audit_user_id`, `audit_user_ip`, `audit_user_info`, `audit_remark`, `audit_date` | TINYINT，注释注明枚举值 |


### 1.4 索引命名规范

**索引名格式**:
- 普通索引：`idx_{表名缩写}_{字段名}`
- 唯一索引：`uk_{表名缩写}_{字段名}`
- 组合索引：`idx_{表名缩写}_{字段1}_{字段2}`

**表名缩写规则**：取各单词首字母，无下划线
- guest_user → gu
- order_info → oi
- order_item → oitem

## 2. 设计流程

### 2.1 从 PRD 提取业务实体

| PRD 位置 | 提取内容 | 示例 |
|----------|---------|------|
| README.md 包含模块 | 各终端模块清单 | 用户中心 → 用户实体 |
| 0-main.md 功能模块 | 模块核心数据对象 | 订单管理 → 订单实体 |
| 模块.md 功能列表 | 每个功能操作的数据 | 商品列表 → 商品实体 |
| 模块.md 页面字段 | 表格/表单字段 | 字段名 → 实体属性 |
| 模块.md 搜索/筛选 | 查询条件 → 索引依据 | 状态筛选 → state 索引 |

### 2.2 覆盖度验证

每个 PRD 功能必须对应至少一个实体，覆盖率 = 100%

| 模块 | 功能 | 依赖实体 | 是否覆盖 |
|------|------|---------|---------|
| 用户中心 | 注册/登录 | guest_user | ✅ |

### 2.3 E-R 关系定义

使用标准符号描述实体关系：
- `||--o{` : 1对多
- `}o--||` : 多对1
- `}o--o{` : 多对多

## 4. 索引设计规则

### 4.1 必须创建索引的场景

- 主键（自动创建）
- 外键字段
- WHERE 条件高频字段
- ORDER BY 字段
- 业务唯一字段

### 4.2 禁止创建索引的场景

- 低基数字段（如性别、布尔值）
- 频繁更新的字段（谨慎）
- 长文本字段（使用前缀索引）

### 4.3 组合索引设计原则

- 遵循最左前缀原则
- 等值查询字段放前面
- 范围查询字段放后面
- 区分度高的字段放前面

### 4.4 索引优化规则

| 规则 | 说明 |
|------|------|
| 覆盖索引 | 查询字段都在索引中，避免回表 |
| 避免函数操作 | `WHERE DATE(create_date) = 'xxx'` 会导致索引失效 |
| 避免隐式转换 | 字段类型与查询值类型必须一致 |
| 控制索引数量 | 单表索引不超过 5-7 个 |

## 5. 非功能设计规范

### 5.1 数据规模评估标准

| 数据量级 | 设计策略 |
|---------|---------|
| < 500万 | 单表 + 索引优化 |
| 500万 - 5000万 | 分区或归档 |
| > 5000万 | 分库分表 |

### 5.2 分库分表策略

| 策略 | 适用场景 | 分片键 |
|------|---------|--------|
| 按租户分库 | SaaS多租户 | saas_id |
| 按时间分表 | 日志、订单 | create_date（按月） |
| 按ID取模 | 用户表 | user_id |
| 按地区分库 | 地域性业务 | region_code |

### 5.3 数据归档规则

| 归档类型 | 触发条件 | 实现方式 |
|---------|---------|---------|
| 定期归档 | 数据超过保留期 | 定时任务迁移到归档表 |
| 冷热分离 | 3个月以上数据 | 热库+冷库分离 |
| 逻辑删除清理 | state=0 且超过保留期 | 物理删除或归档 |

### 5.4 安全规范

| 安全措施 | 实现要求 |
|---------|---------|
| 敏感字段加密 | 手机号、身份证 AES 加密 |
| 密码存储 | BCrypt 哈希 + 盐 |
| 数据脱敏 | 日志中敏感信息脱敏 |
| 访问控制 | 最小权限原则 |

### 5.5 高可用要求

- 主从复制架构
- 全量备份：每周一次
- 增量备份：每日一次
- Binlog 备份：实时

## 6. 设计原则

| 原则 | 说明 |
|------|------|
| 第一范式 | 字段原子性，不可再分 |
| 第二范式 | 消除部分依赖 |
| 第三范式 | 消除传递依赖 |
| 反范式化 | 适当冗余减少关联查询（谨慎使用） |

---

## 7. 约束规范

- **禁止使用外键约束**，由程序控制外键关系
- 所有表必须包含 `saas_id` 字段（SaaS架构）
- 所有表必须包含 `create_date`, `modify_date`, `state` 通用字段
- 金额/价格使用 BIGINT 类型，单位为分
- 状态字段使用 TINYINT，注释中说明枚举值含义
- 每个字段必须有 COMMENT 注释
- 表注释格式为"模块-实体"
