# 测试设计评审检查清单

## 1. API 接口测试评审

| 检查项 | 要求 |
|--------|------|
| 接口覆盖 | 所有Controller接口均有对应用例 |
| 参数校验 | 必填/选填、类型、长度、格式均有验证用例 |
| 状态码覆盖 | 200/400/401/403/404/500 均有覆盖 |
| 权限矩阵 | 每个角色×接口的权限验证 |
| 异常处理 | 空值、超长、特殊字符、并发场景 |
| 数据驱动 | 参数化测试数据独立列出 |
| 请求/响应格式 | 请求参数和预期响应JSON完整定义 |

## 2. E2E 界面测试评审

| 检查项 | 要求 |
|--------|------|
| 页面覆盖 | 所有页面/路由均有导航测试 |
| 操作覆盖 | CRUD操作、列表交互、状态变更 |
| 元素定位 | 基于字段一致性原则，优先使用 role/label/placeholder/name 属性定位（无需 data-testid） |
| 跨终端流程 | 多角色协作流程有独立跨终端用例 |
| 角色覆盖 | 每个角色的可见性和操作权限验证 |
| 优先级标记 | P0/P1/P2/P3 分级明确 |
| 测试数据 | 独立列出，不内嵌在步骤中 |

## 3. 压力测试评审

| 检查项 | 要求 |
|--------|------|
| 场景类型 | 基准/负载/压力/稳定性四种场景齐全 |
| SLA指标 | P95/P99/错误率/TPS目标值合理且有依据 |
| 线程组配置 | 线程数、ramp-up、持续时间参数合理 |
| 参数化数据 | Token提取方式、CSV数据文件定义清晰 |
| 响应断言 | HTTP状态码 + 业务code + 响应时间断言 |
| 监听器 | 结果采集方式（Aggregate Report/Backend Listener） |
| 思考时间 | 模拟真实用户操作间隔 |

## 4. 安全测试评审

| 检查项 | 要求 |
|--------|------|
| OWASP Top 10 | 每个类别均有对应检查项 |
| SQL注入 | 所有含输入参数的接口有注入测试 |
| XSS | 反射型/存储型/DOM型均有覆盖 |
| 越权访问 | 水平越权、垂直越权、IDOR |
| 认证授权 | Token验证、暴力破解防护、Session管理 |
| 敏感数据 | 响应脱敏、密码哈希、日志脱敏 |
| CSRF | Token验证、SameSite Cookie配置 |
| 依赖漏洞 | Trivy扫描覆盖后端/前端/镜像 |

## 5. 通用质量检查

| 检查项 | 要求 |
|--------|------|
| 用例ID规范 | `TC-{类型}-{模块}-{序号}` 格式统一 |
| 前置条件 | 登录角色、初始数据明确 |
| 可自动化性 | 每个步骤可映射为 Playwright/JMeter 操作 |
| 独立性 | 每个用例独立可执行，无强依赖 |
| 可重复性 | 明确测试数据，多次执行结果一致 |
