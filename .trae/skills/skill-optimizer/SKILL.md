# Skill Optimizer - SKILL.md 优化器

## 描述

专门优化 SKILL.md 文件的技能，通过三层架构设计（Metadata → 主体 → References）实现渐进式内容展示，减少 token 消耗，提升 AI 使用效率。

## 使用场景

| 触发条件        | 示例                     |
| ----------- | ---------------------- |
| 优化现有 skill  | "优化这个 skill 文件"        |
| 检查规范符合度     | "检查 skill 是否符合规范"      |
| 精简文件大小      | "精简 skill 文件，减少 token" |
| 提升使用效率      | "让这个 skill 更高效"        |
| 重构 skill 结构 | "重构 skill 目录结构"        |

## 优化原则

### 三层架构设计

| 层级         | 内容                 | 建议大小             | 存储位置           |
| ---------- | ------------------ | ---------------- | -------------- |
| Metadata   | name + description | \~100 词          | SKILL.md 头部    |
| 主体         | 核心工作流程             | <=300 行，<=5000 词 | SKILL.md       |
| References | 详细文档/模板/示例         | 无限制              | references/ 目录 |

### 精简公式

```
SKILL.md = 核心流程 + 选择指导 + 资源引用
```

| 保留在 SKILL.md | 移到 references/ | 移到 scripts/ |
| ------------ | -------------- | ----------- |
| 核心流程         | 变体细节、示例代码      | 可执行代码       |
| 触发条件         | 配置选项、API 文档    | 自动化脚本       |
| 选择指导         | 详细案例           | 数据处理脚本      |

## 指令

### 第一步：分析当前 Skill

1. **读取目标文件**：`{skill-path}/SKILL.md`
2. **统计文件指标**：
   - 总行数
   - 总词数（估算）
   - description 长度
   - 表格数量
   - 代码块数量
3. **识别问题**：
   - ❌ description > 500 字符
   - ❌ 文件 > 300 行
   - ❌ 包含通用 AI 能力描述
   - ❌ 包含普遍最佳实践（DRY、SOLID）
   - ❌ 自明的代码信息
   - ❌ 弱指令（"尽量"、"应该"）

### 第二步：优化 Metadata

**description 优化公式**：

```
技能功能 + 使用场景 + 具体触发条件（数字编号）
```

| 优化前（❌）    | 优化后（✅）                                                                   |
| --------- | ------------------------------------------------------------------------ |
| "PDF编辑技能" | "综合PDF编辑、转换和文本提取技能。当Claude需要处理PDF文件时使用：(1)编辑内容，(2)转换格式，(3)提取文本，或其他PDF任务" |
| "代码审查技能"  | "Java代码质量审查技能。当需要审查Java代码时触发：(1)检查代码规范，(2)识别潜在bug，(3)评估性能问题"             |

**规则**：

- 长度 ≤ 500 字符
- 包含 3-5 个具体触发条件
- 使用数字编号

### 第三步：精简主体内容

#### 1. 使用表格替代段落

| 信息类型 | 优化方式           |
| ---- | -------------- |
| 配置项  | 表格列举           |
| 参数说明 | 表格 + 类型/必填/默认值 |
| 错误码  | 表格：错误码/含义/解决方案 |
| 角色职责 | 表格：角色/职责/对应智能体 |
| 技术栈  | 表格：组件/版本/用途    |

#### 2. 删除冗余内容

**坚决删除**：

- 通用 AI 能力（"写干净代码"、"调试有效"）
- 普遍最佳实践（DRY、SOLID、KISS）
- 自明的代码信息（使用的编程语言）
- 父级已定义的规则
- 过时的文件路径

**保留**：

- 量化约束（"函数最多 50 行"）
- 项目特定架构偏差
- 明确禁止事项（"永不使用库 X"）
- 领域术语

#### 3. 弱指令转强指令

| 弱指令（❌）     | 强指令（✅）                             |
| ---------- | ---------------------------------- |
| "尽量保持函数小"  | "函数 >50 行需添加合理性注释"                 |
| "使用好的命名规范" | "布尔变量：is/has/should 前缀。函数：动词-名词"   |
| "注意性能"     | "循环内避免数据库查询，批量操作单次 ≤1000 条"        |
| "处理好错误"    | "所有外部调用必须 try-catch，记录 error 级别日志" |

### 第四步：创建 References 目录

如果 SKILL.md 超过 300 行，创建分层目录：

```
{skill-name}/
├── SKILL.md              # 主入口，精简，<300 行
├── references/
│   ├── template.md       # 输出模板
│   ├── examples.md       # 详细示例
│   └── advanced.md       # 高级用法
└── scripts/              # （可选）
    └── helper.py         # 辅助脚本
```

**迁移内容清单**：

| 原位置    | 迁移目标                     | 引用方式                                    |
| ------ | ------------------------ | --------------------------------------- |
| 详细示例代码 | `references/examples.md` | "详见 [examples](references/examples.md)" |
| 输出模板   | `references/template.md` | "使用 [模板](references/template.md)"       |
| 配置选项详解 | `references/config.md`   | "配置详情见 [config](references/config.md)"  |
| API 文档 | `references/api.md`      | "API 参考 [api](references/api.md)"       |
| 长代码块   | `scripts/`               | "执行 `scripts/run.py`"                   |

### 第五步：验证优化结果

**检查清单**：

- [ ] description ≤ 500 字符
- [ ] description 包含具体触发条件
- [ ] SKILL.md ≤ 300 行（理想 <200 行）
- [ ] 使用表格替代了长段落
- [ ] 删除了通用 AI 能力描述
- [ ] 指令具体可执行（无量化词）
- [ ] references/ 目录已创建（如需要）
- [ ] 文件引用路径正确
